**版本**: 1.0 | **日期**: 2026-05-15
**定位**: 竞品雷达 + 政策哨兵 + 趋势探测器

## 一、三线采集框架

```
市场数据层
├── 政策哨兵线 🛡️  (policy_monitor → policy_snapshots → policy_updates)
│   ├── S级: 入境处/劳工局/税务局/教育局 .gov.hk 官方页面
│   ├── 采集频率: 每日Cron + 手动触发
│   └── 交付: 政策简报 + 影响评估
│
├── 竞品雷达线 📡  (小红书采集 + 应用商店监测)
│   ├── B/C级: 竞品小程序功能/定价/评价
│   ├── 采集频率: 每两周手动/2条
│   └── 交付: 竞品月报
│
└── 趋势探测线 🔭  (行业报告 + 小红书内容趋势 + 微信指数)
    ├── A/B级: 行业报告/媒体报道
    ├── 采集频率: 每月手动/3条
    └── 交付: 趋势周报/月报
```

## 五、市场数据采集SOP

### 5.1 政策采集 (policy_monitor Cron - 待Claude配置)

```
每日 09:00 → policy_monitor 触发
  → 扫描5个S级来源域名
  → 对比 policy_snapshots 上一条快照
  → 有变化 → 生成 diff + policy_updates + 通知
  → 无变化 → 更新 capturedAt
```

### 5.2 竞品采集 (手动 + BI工具辅助)

```
每两周:
  1. 打开各竞品小程序 → 截图功能/定价页
  2. 微信指数查询竞品关键词
  3. 小红书搜索竞品名称 → 汇总用户反馈
  4. 更新竞品月报
```

### 5.3 趋势采集 (每月 + 事件驱动)

```
每月:
  1. 微信指数查询3-5个核心关键词
  2. 百度指数补充
  3. 小红书热门话题扫描

事件驱动:
  - 政府公布年度《人才清单》 → 即时扫描
  - 重大政策发布 → 即时通知
```

## 七、数据治理

- 所有采集内容遵循 data-collection-governance skill
- 竞品信息标 C+ 级 (仅供内部参考,不对外)
- 政策信息标 S/A 级

- 所有分析附带来源URL和采集日期
