# 双闸门方案 — 回归风险评估

> 2026-05-21 | 评估在现有代码中加双闸门（登录态 + 身份状态）的回归风险

## 一、高风险回归点（P0 — 会静默绕过闸门）

### 1. `selectTemplate` 绕过 + 强制覆写 userStatus

`pages/process/index/index.js:683-722`

当前行为：skipped 用户点击「选择流程路径」→ 模板面板 → 选择模板 → `selectTemplate` 执行 → 创建流程线，并将 `userStatus` 强制改为 `'unapplied'`（line 713）。用户从 skipped 变成了 unapplied，且完全无声。

回归风险：闸门加在 `onSelectDirectPath` 上，但 `selectTemplate` 是另一条路径创建入口。不在此加门禁 = 闸门形同虚设。更严重的是 `userStatus = 'unapplied'` 会覆盖用户真实状态。

```
selectTemplate: {
  创建流程线 ✓
  设置 selectedPath ✓
  强制 userStatus = 'unapplied' ← 用户从 skipped 变成 unapplied，无声无息
  无任何 isLoggedIn 检查 ✓
  无任何 userStatus 检查 ✓
}
```

### 2. `floating-ai` 组件 `selectPathFromChat` 直接赋值

`components/floating-ai/floating-ai.js:220-236`

当前行为：AI 对话推荐路径后，用户点击采纳 → `selectPathFromChat(pathType, label)` → 直接 `app.globalData.selectedPath = pathType`。仅检查 `isLoggedIn` 用于是否同步云端，不阻断赋值。

回归风险：访客或 skipped 用户通过 AI 对话获得路径推荐后可以直接设置 selectedPath。虽然不会创建流程线，但设置了路径相当于做了"半决策"。后续其他页面读到 selectedPath 后会按有路径的用户展示功能。

### 3. 评估结果页 `selectPath`

`subpkg-low/pages/assessment-result/index.js:131`

当前行为：`app.globalData.selectedPath = path`，无任何门禁检查。

回归风险：同 floating-ai，路径被设置但没有对应流程线和身份状态。

## 二、中风险回归点（P1 — 影响用户体验）

### 4. 流程控 skip-banner 与闸门的交互

当前：`isSkipped` 横幅展示「请先选择身份状态」+「去选择」按钮——此按钮是软引导（navigateTo status-select），用户可以忽略横幅继续操作下方的路径卡片。

加闸门后：横幅下方的路径卡片被门禁阻断，用户点击路径卡片时弹出登录/身份引导 Sheet。横幅 + Sheet 同时存在可能让用户困惑。

建议：加闸门后，skipped 用户看不到路径选择按钮（仅保留横幅引导），避免双重提示。

### 5. 访客浏览路径信息的体验

当前：未登录用户可以在流程控看到路径选项（名称、图标、描述），只是点击后直接创建流程线（Bug）。

加闸门后：路径信息展示不变（L0 浏览层），但点击后弹出登录半屏。对于只是在了解路径信息的用户，这个"打断"可能不被接受——他们没想做决策，只是想看看。

建议：路径卡片区分"查看"和"选择"两种交互。点击卡片展开详情（无需登录），详情底部「选择此路径」按钮触发闸门。

### 6. status-badge 路径切换入口

当前：`onTapPath()` → `navigateTo path-select`。skipped 用户点击路径标签后进入 path-select 页。

加闸门后：path-select 的 `onSelect` 被门禁阻断。但用户点击路径标签的意图是"切换路径"——如果 skipped 状态没有旧路径，切换到新路径等价于首次选择，应该通过闸门。如果已有旧路径且想切换，也应该通过闸门。门禁逻辑正确，但需要确保半屏引导文案准确。

## 三、低风险回归点（P2 — 极端情况）

### 7. 存储损坏导致 userStatus 不一致

场景：Storage 中 `__user_status__` = 'unapplied'，但 `__session__` 被清除导致 `isLoggedIn = false`。

当前行为：`stage-helper.js:53` 会读 `__user_status__` 判断进度。闸门检查 `isLoggedIn` 优先——未登录直接拒绝，逻辑正确。

### 8. ¥599 重置后 skipIdentity 的串联

场景：用户支付 ¥599 → 存储清空 → 跳转 status-select → skipIdentity → 回流程控。

当前行为：所有数据已清除，userStatus='skipped'，无流程线。闸门检查：`isLoggedIn=true`（登录态未丢失），`userStatus='skipped'` → 闸门2失败 → 弹出身份确认引导。逻辑正确，无回归。

### 9. selectTemplate 中 **process_stage** = 0 的问题

当前：`selectTemplate` 将 `__process_stage__` 设为 0（line 716），而 `onSelectDirectPath` 设为 1（line 195）。两者不一致。

加闸门后：skipped 用户无法执行 `selectTemplate`（被闸门阻断），所以 0 vs 1 的差异不会在 skipped 用户身上触发。但建议统一——如果 `selectTemplate` 也要自动完成 phase1，应设为 1。

## 四、结论 — 最少需加门禁的位置

```
必须加门禁的 5 个入口（P0）：
  1. process/onSelectDirectPath   — 路径卡片选择
  2. process/selectTemplate       — 模板流程创建（额外风险：覆写 userStatus）
  3. path-select/onSelect         — 独立页面路径选择
  4. assessment-result 选路径处    — AI 评估结果选路径
  5. floating-ai/selectPathFromChat — AI 对话推荐选路径

建议加门禁的 1 个入口（P1）：
  6. floating-ai 首条 AI 消息      — 非决策但可引导登录以获得个性化回复

无需加门禁的入口：
  - 攻略书浏览/阅读关卡内容
  - 证件夹空状态浏览
  - 提醒器空状态浏览
  - 我的页面设置/客服
  - AI 对话（仅做推荐不绑路径）
  - 攻略书关卡任务操作（本身有解锁门禁）
```
