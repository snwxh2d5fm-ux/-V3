#### P1-02: 全部 6 个 admin 云函数存在空 catch 块

| 严重度 | 🟠 P1 |
|:--|:--|
| 规则 | 编码规范 #1：禁止空 catch 块 |
| 影响范围 | 6 个 admin 云函数全部 |
| 位置 | 每个函数的 JSON.parse 处: `try { body = JSON.parse(event.body); } catch (_) {}` |
| 风险 | JSON 解析失败时静默跳过，可能传入非标准 body 导致后续逻辑异常且无法追踪 |
| 修复建议 | 替换为:

```js
try { body = JSON.parse(event.body); } catch (e) {
  console.error('[admin-xxx] JSON parse error:', e.message, event.body?.slice(0, 100));
  return { code: 400, msg: '请求格式无效' };
}
```
| 工作量 | 6 处修改，每处约 5 行 |

#### P1-03: Math.random() 用于邀请码生成（安全敏感）

| 严重度 | 🟠 P1 |
|:--|:--|
| 规则 | 编码规范 #8：禁止 Math.random() 用于安全逻辑 |
| 文件 | `cloudfunctions/invite-code/index.js` |
| 行号 | L665 |
| 代码 | `raw += CODE_CHARS[Math.floor(Math.random() * CODE_CHARS.length)];` |
| 风险 | `Math.random()` 不是密码学安全随机数。邀请码作为年卡会员凭证，可被预测后暴力枚举，导致未授权激活 |
| 修复建议 | 替换为:

```js
const getRandomIndex = (max) => {
  const buf = crypto.randomBytes(4);
  return buf.readUInt32BE(0) % max;
};
raw += CODE_CHARS[getRandomIndex(CODE_CHARS.length)];
```
| 工作量 | 生成函数内 5-8 行改造 |
| 注意 | 当前使用 `wx-server-sdk`，需确保 `crypto` 模块在云函数环境可用 |
