## 二、新增文件

### 2.1 `utils/decision-gate.js` — 闸门引擎

```javascript
/**
 * 住港伴 V4 — 双闸门决策引擎
 * 产品方案 v2.0: 决策 = 已登录 + 已确认身份状态
 */

var VALID_STATUSES = ['unapplied', 'submitted', 'approved', 'permanent'];

/**
 * 检查当前用户是否可以做"决策"（选路径/创建流程）
 * @returns {{ ok: boolean, reason: 'login'|'identity'|null }}
 *   ok=false, reason='login'   → 闸门1失败：未登录
 *   ok=false, reason='identity' → 闸门2失败：已登录但未确认身份
 *   ok=true                     → 双闸门通过
 */
function canMakeDecision() {
  var app;
  try {
    app = getApp();
  } catch (e) {
    app = null;
  }

  // 闸门1: 登录态
  var isLoggedIn = !!(app && app.globalData && app.globalData.isLoggedIn);
  if (!isLoggedIn) return { ok: false, reason: 'login' };

  // 闸门2: 身份状态已确认（非空、非 skipped）
  var userStatus = app.globalData.userStatus || wx.getStorageSync('__user_status__') || '';

  if (!userStatus || userStatus === 'skipped') {
    return { ok: false, reason: 'identity' };
  }

  return { ok: true, reason: null };
}

module.exports = { canMakeDecision: canMakeDecision };
```

**设计决策：**

- 读 `globalData.userStatus` 为主，`Storage.__user_status__` 为兜底——同现有 `stage-helper.js` 模式
- 不调用云函数（同步判定，零延迟）——登录态和身份状态均由本地维护
- 不检查 `isLocked`——锁定是支付/会员维度，不影响身份决策

### 2.2 `components/gate-sheet/` — 引导半屏组件

```
components/gate-sheet/
├── gate-sheet.js    — 组件逻辑
├── gate-sheet.wxml  — 模板
├── gate-sheet.wxss  — 样式
└── gate-sheet.json  — 组件声明
```

**组件属性：**

| 属性        | 类型    | 说明                         |
| ----------- | ------- | ---------------------------- |
| `show`      | Boolean | 是否显示半屏                 |
| `mode`      | String  | `'login'` / `'identity'`     |
| `pathLabel` | String  | 待选路径名称（用于文案展示） |

**组件事件：**

| 事件          | 触发时机                                     | detail |
| ------------- | -------------------------------------------- | ------ |
| `gate-passed` | 登录成功 或 身份已确认（status-select 回调） | `{}`   |
| `dismiss`     | 用户点击"暂不"/"稍后再说"                    | `{}`   |

**login 模式内部流程：**

```
用户点击「微信一键登录」
  → wx.login() 获取 code
  → wx.cloud.callFunction('user-auth', { action:'login', code })
  → 云函数返回 token + userStatus
  → app.saveSession() 持久化
  → 检查闸门2: userStatus 是否有效
     → 有效 → triggerEvent('gate-passed')
     → skipped/空 → 切换 mode='identity'（登录成功但需确认身份）
```

**identity 模式内部流程：**

```
用户点击「去确认身份」
  → wx.navigateTo('/pages/status-select/status-select')
  → 用户完成身份选择后返回
  → 组件在 pageLifetimes.show 中重新检查闸门
  → 有效 → triggerEvent('gate-passed')
  → 仍 skipped → 不做任何事（用户又跳过了）
```

**关键设计决策：**

- 组件自包含登录逻辑——不依赖外部调用方的登录处理
- 切换 identity 模式复用同一组件——避免两套 UI
- 半屏底部弹出（`position:fixed; bottom:0; z-index:1000`），不阻断页面渲染

**闸门通过后的回调：**

```javascript
onGatePassed: function() {
  var id = this.data.pendingPathId;
  var label = this.data.pendingPathLabel;
  this.setData({ showGateSheet: false, pendingPathId: '', pendingPathLabel: '' });
  if (id) {
    // 重新触发 onSelectDirectPath（此时闸门已通过）
    this.onSelectDirectPath({ currentTarget: { dataset: { id: id } } });
  }
}
```

**数据新增：**

```javascript
data: {
  // ... 原有
  showGateSheet: false,     // 闸门半屏显示
  gateMode: '',             // 'login' | 'identity'
  pendingPathId: '',        // 待执行的路径ID
  pendingPathLabel: ''      // 待执行的路径名称
}
```

### 3.5 `components/floating-ai/floating-ai.js` — 组件内部

```javascript
var gate = require('../../utils/decision-gate').canMakeDecision();
if (!gate.ok) {
  // 组件内无法弹 Sheet，改为 Toast + 跳转
```

## 四、数据流

```
用户点击决策入口
       │
       ▼
canMakeDecision()
       │
   ┌───┴───┐
   │ ok?   │
   └───┬───┘
  Yes  │  No
   ↓   │   ↓
 执行  │  showGateSheet = true
 原逻辑 │  gateMode = reason
       │       │
       │  ┌────┴────┐
       │  │ login   │  identity
       │  │ 模式    │  模式
       │  └────┬────┘
       │       │
       │   gate-passed 事件
       │       │
       │  onGatePassed()
       │       │
       │  重新触发原决策入口
       │  （此时闸门通过）
       │       │
       └───────┘
         正常执行
```

**不修改 `app.globalData` 结构**——`isLoggedIn` 和 `userStatus` 已存在，`decision-gate` 只读不写。

**不修改 `skipLogin()` 和 `skipIdentity()`**——这两个函数的行为保持不变，闸门在决策点拦截，不是入口拦截。
