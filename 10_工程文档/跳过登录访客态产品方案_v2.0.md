# 访客态与身份状态 — 双闸门产品方案 v2.0

> 2026-05-21 | v1.0 修正版 — 区分"身份状态"与"身份路径规划"

## 一、概念澄清

产品中有两个独立概念，不可混淆：

| 概念             | 键                             | 含义                                 | 选项                                                                                                                                                           |
| ---------------- | ------------------------------ | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **身份状态**     | `userStatus` / `userSubStatus` | 用户在香港身份旅程中所处的阶段和角色 | `unapplied`(在校/在职/企业主/海外华人)、`submitted`(优才/高才通/专才/IANG/CIES)、`approved`(在港就业/创业/学习/内地)、`permanent`(已永居)、`skipped`(暂不选择) |
| **身份路径规划** | `selectedPath`                 | 用户计划通过什么路径进行身份规划     | `qmas`、`ttps_a/b/c`、`asmpt`、`student_iang`、`dependent`、`cies`、`techtas`、`retirement` 等13条路径                                                         |

**核心规则：用户做"决策"时，必须同时满足两个条件 — 已登录 + 已确认身份状态。**

```
决策门禁 = isLoggedIn && userStatus !== 'skipped' && userStatus !== null
```

## 三、决策点清单（需通过双闸门）

以下操作触发路径选择/流程创建，必须两个闸门都通过：

| #   | 决策点                           | 位置                               | 当前门禁 |
| --- | -------------------------------- | ---------------------------------- | :------: |
| 1   | 流程控「直接选择路径」           | `process/index onSelectDirectPath` |  ❌ 无   |
| 2   | 流程控「AI智能评估」→ 结果选路径 | `assessment-result selectPath`     |  ❌ 无   |
| 3   | 独立路径选择页                   | `path-select/index onSelect`       |  ❌ 无   |
| 4   | AI对话中推荐路径并选择           | `floating-ai handleSelectPath`     |  ❌ 无   |

## 四、决策点门禁实现

```
function canMakeDecision() {
  var app = getApp();
  // 闸门1: 必须登录
  if (!app.globalData.isLoggedIn) return { ok: false, reason: 'login' };
  // 闸门2: 必须已确认身份状态
  var us = app.globalData.userStatus || wx.getStorageSync('__user_status__') || '';
  if (!us || us === 'skipped') return { ok: false, reason: 'identity' };
  return { ok: true };
}
```

### 4.1 闸门1 失败（未登录）→ 登录引导半屏

弹出底部 Sheet：「登录后开启专属流程」→ 微信一键登录 → 登录成功后检查闸门2。

### 4.2 闸门2 失败（已登录但身份 skipped）→ 身份确认引导

弹出底部 Sheet：「请先确认你的身份状态」→ 两个按钮：

- 「去确认身份」→ `navigateTo status-select`
- 「稍后再说」→ 关闭 Sheet，决策被阻断

### 4.3 双闸门通过 → 正常执行原逻辑

## 五、"暂不选择身份"后的回归引导点

用户从 `status-select` 点击 `skipIdentity()` → `userStatus='skipped'` → `reLaunch` 到流程控。以下场景应引导用户回归选择身份状态：

### 5.1 已存在的回归点（当前代码已实现）

| 页面             | 触发条件                   | UI                                               | 按钮                                 |
| ---------------- | -------------------------- | ------------------------------------------------ | ------------------------------------ |
| 流程控 process   | `isSkipped === true`       | `skip-banner` 横幅「请先选择身份状态」           | 「去选择」→ `goSelectIdentity()`     |
| 证件夹 documents | `isSkipped === true`       | `skip-banner` 横幅「请先选择身份后解锁完整功能」 | 「去选择」→ `goSelectIdentity()`     |
| 旧首页 index     | `userStatus === 'skipped'` | hero 空状态卡片                                  | 「去选择身份」→ `goSelectIdentity()` |

这三处已覆盖主要入口。**无需新增回归点。**

### 5.2 需要强化的场景

| 场景                                | 当前行为                  | 建议                                   |
| ----------------------------------- | ------------------------- | -------------------------------------- |
| skipped 用户在流程控点路径卡片      | **直接创建流程线（Bug）** | 触发闸门2失败 → 弹出身份确认引导 Sheet |
| skipped 用户在 path-select 页选路径 | **直接创建流程线（Bug）** | 同上                                   |

### 5.3 无需回归的场景

- 攻略书浏览 → skipped 用户可正常浏览所有攻略内容
- 提醒器查看 → 无路径时自然空状态
- 我的页面 → skipped 用户可看到状态提示
- AI 对话 → 允许使用，但做路径相关推荐时不自动绑定

## 六、已登录但未选身份（skipped）与 访客态 对比

| 维度       | 访客态 (!isLoggedIn) | skipped (isLoggedIn + userStatus='skipped') |
| ---------- | -------------------- | ------------------------------------------- |
| 浏览攻略书 | ✅                   | ✅                                          |
| 浏览证件夹 | ✅ 空状态            | ✅ skip-banner + 空状态                     |
| 选中路径   | ❌ 闸门1拦截         | ❌ 闸门2拦截                                |
| AI对话     | ✅ 无上下文          | ✅ 可提供身份上下文（如有）                 |
| 云端同步   | ❌                   | ✅                                          |
| 反馈提交   | ✅                   | ✅                                          |
| 引导回归   | 登录半屏             | 身份确认半屏                                |
