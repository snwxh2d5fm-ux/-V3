# 预存缺陷专项方案 v1.0

## 背景

真机测试暴露 `tracker.track is not a function` 崩溃，根因为 `4aea8c9` (V4.2) 中引入的API名称错误（调用不存在的 `track()` 而非 `event()`）。此类缺陷的共同特征：**代码写入了但执行路径从未被触发**，导致静态检查和常规测试均无法捕获。

全项目扫描后，共发现 **5项预存缺陷**（2项已修复，2项待修复，1项待清理）。

## 三、预防机制

### 3.1 ESLint 规则（立即可行）

在项目 `.eslintrc` 中启用：

```json
{
  "rules": {
    "no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }],
    "import/no-unresolved": "error",
    "node/no-missing-require": "error"
  }
}
```

安装 `eslint-plugin-import` 和 `eslint-plugin-node`：

```
npm install --save-dev eslint-plugin-import eslint-plugin-node
```

新增自定义脚本：扫描所有 require 路径是否存在对应的文件。

### 3.2 静态扫描脚本 `scripts/audit-requires.js`（新建）

```javascript
// 扫描项目所有 require() 调用，验证：
// 1. 目标文件/目录存在
// 2. 调用的方法在 module.exports 中存在
// 输出: JSON 格式的缺陷报告
```

纳入 `package.json` scripts：
```json
"audit:requires": "node scripts/audit-requires.js"
```

### 3.3 pre-commit Hook

在 `.husky/pre-commit` 中增加：

```bash
#!/bin/sh
npm run audit:requires
```

### 3.4 真机测试补充规则（流程层面）

在 `wechat-miniprogram-upload-checklist` 中增加：

```
- [ ] 新建流程 → 选路径 → 确认不崩溃
- [ ] 自评 → 选路径 → 确认不崩溃
- [ ] 文档 → 裁剪确认 → 确认不崩溃
```

列为必测项，每次提审前执行。
