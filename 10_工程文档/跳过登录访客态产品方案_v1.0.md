# 跳过登录访客态产品方案 v1.0
> 2026-05-21 | 针对流程控未登录可选路径的问题

## 一、问题定义

用户从首页欢迎页 → 其他方式登录 → 登录页点击"跳过" → 直接进入攻略书Tab。此时处于**未登录匿名访客态**，但流程控Tab允许直接选择身份路径并推进阶段，核心功能绕过了登录门禁。同时存在两个衍生问题：

- `isSkipped` 状态逻辑是**死代码**（`process/index.js:79` 和 `documents/index.js:127` 都检查 `userStatus === 'skipped'`，但 `skipLogin()` 从未写入此状态）
- 无任何机制将访客引导回登录流程

## 二、产品原则

| 原则 | 说明 |
|------|------|
| **渐进式信任** | 访客可预览功能价值，但核心操作需登录——不靠拦截，靠价值吸引 |
| **不阻断体验** | 禁止弹窗打断用户当前操作，只在用户主动发起受限操作时引导 |
| **状态一致性** | 访客态必须有明确标记，与已登录用户的状态不可混淆 |
| **数据隔离** | 访客数据仅存本地，不触发云端同步，避免脏数据 |

## 四、流程控选路径 — 核心方案

### 4.1 当前行为 vs 目标行为

| 场景 | 当前行为 | 目标行为 |
|------|---------|---------|
| 已登录用户选路径 | 点击即创建流程线 | 不变 |
| 访客点击路径卡片 | **直接创建流程线（Bug）** | 弹出登录引导半屏，登录后自动创建 |
| 访客点击"一触即选" | 直接创建（Bug） | 同上 |

### 4.2 登录引导半屏设计

访客在流程控点击任意路径卡片时：

1. 弹出底部半屏（Sheet），非阻断式Modal
2. 半屏内容：
   - 标题：「登录后开启专属流程」
   - 副标题：已选路径 "{路径名称}" 将在登录后自动创建
   - 两个按钮：
     - 主按钮「微信一键登录」→ 调用 `wx.login` → `user-auth` 云函数 → 登录成功后自动 `onSelectDirectPath`
     - 次按钮「暂不登录」→ 关闭半屏，停留在路径列表页
3. 半屏关闭后，用户仍在流程控页面，可继续浏览路径信息

### 4.3 技术实现要点

```
onSelectDirectPath(e) 入口改动：
  if (!app.globalData.isLoggedIn) {
    this.setData({ 
      showLoginSheet: true, 
      pendingPathId: id, 
      pendingPathLabel: label 
    });
    return;  // ← 阻断选路径
  }
  // ... 原有逻辑不变

handleLoginFromSheet()  →  登录成功后读取 pendingPathId → 执行 onSelectDirectPath 原逻辑
handleDismissSheet()    →  仅关闭半屏，不创建流程，不写任何状态
```

## 五、全链路登录引导点

登录引导不应该是"门禁拦截"，而应该是"价值引导"。以下是各Tab的引导策略：

### 5.1 攻略书Tab
- **无需改动**。关卡内容可浏览，锁定关卡本身就是动机——用户看到灰色关卡自然想解锁。
- 锁定关卡的任务checkbox点击时，当前已有Toast「请先解锁本关卡」→ 增加一行「登录后推进流程可解锁」

### 5.2 证件夹Tab  
- 访客看到空状态页，展示功能介绍
- 底部增加「登录后开始管理证件」引导按钮
- 点击按钮 → 跳转登录页（带回调参数，登录后返回证件夹）

### 5.3 提醒器Tab
- 访客看到功能介绍+示例提醒
- 底部「登录后自动生成提醒」引导按钮
- 同证件夹模式

### 5.4 流程控Tab
- **核心改动**：路径选择触发登录半屏（见第四节）
- 阶段推进按钮（完成阶段/上传验证）本身已有 `activeProcessId` 校验——访客无流程线，自然无法操作

### 5.5 我的Tab
- 当前已有登录状态展示
- 增加「一键登录」入口在页面顶部（当前已有，确认是否可见）

## 六、skipLogin 状态修复

当前 `isSkipped` 是死代码，需做出选择：

**方案A（推荐）：废弃 `isSkipped`**
- 删除 `process/index.js:79` 和 `documents/index.js:127` 中的 `isSkipped` 逻辑
- 访客态用 `!app.globalData.isLoggedIn` 统一判断
- `skipLogin()` 不需要写任何状态——它只是跳转

**方案B：激活 isSkipped**
- `skipLogin()` 时写入 `userStatus = 'skipped'`
- 各页面根据 `isSkipped` 展示引导按钮
- 增加状态管理复杂度

推荐方案A，因为 `isLoggedIn` 已经是足够的状态标记。

## 八、验收标准

- [ ] 访客在流程控点击路径 → 弹出登录半屏，不创建流程线
- [ ] 半屏中点击「微信一键登录」→ 登录成功 → 自动创建所选路径流程线
- [ ] 半屏中点击「暂不登录」→ 关闭半屏，留在流程控页面，不写任何数据
- [ ] 已登录用户选路径 → 行为不变（无回归）
- [ ] 证件夹/提醒器访客态展示引导按钮
- [ ] `isSkipped` 死代码清理完成
