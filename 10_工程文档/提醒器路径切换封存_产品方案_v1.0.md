# 提醒器路径切换封存 — 产品方案 v1.0

> 2026-05-21 | 优先级 P2 | 1 SP

## 一、问题

用户从"优才"切换至"高才通A类"后，提醒器列表混合展示两套路径的提醒。虽已有 `pathway` 字段过滤当前路径，但用户无感知——不知道旧提醒被隐藏，也不知道切回去能恢复。

## 二、方案

**切换路径时，旧路径提醒自动封存；切回原路径时自动解封。**

### 2.1 触发时机

| 触发             | 位置                         |
| ---------------- | ---------------------------- |
| 流程控直接选路径 | `process/onSelectDirectPath` |
| 流程控模板选路径 | `process/selectTemplate`     |
| 独立路径选择页   | `path-select/onSelect`       |

### 2.2 封存逻辑

```
切换路径时:
  oldPath = 切换前的 selectedPath
  if oldPath 存在且 ≠ 新路径:
    所有 pathway === oldPath 且 status === 'active' 的提醒
    → status = 'archived'

切回原路径时:
  所有 pathway === 新路径 且 status === 'archived' 的提醒
  → status = 'active'
```

### 2.3 显示规则

| 提醒状态                         | 展示位置                                            | 视觉 |
| -------------------------------- | --------------------------------------------------- | ---- |
| `active` + 当前路径              | 主列表                                              | 正常 |
| `active` + 无pathway（手动提醒） | 主列表                                              | 正常 |
| `archived`                       | 列表底部折叠区「已封存 N 条（切换回原路径后恢复）」 | 灰色 |
| `completed`                      | 已完成筛选                                          | 不变 |

### 2.4 不受影响的提醒

- 手动提醒（`pathway` 为空）——始终显示，不封存
- 已完成提醒（`status === 'completed'`）——不封存
- 已忽略提醒（`status === 'ignored'`）——不封存

## 四、产品决策记录

- **证件夹路径关联**：暂不实施。原因：身份证明类文档（港澳通行证、身份证等）天然跨路径通用，需先完成文档分类规则定义后才能实施，否则会误隐藏通用证件。
- **提醒器封存恢复**：切回原路径自动解封，无需用户手动操作。封存的提醒保留原 `deadline`，不会因封存期间时间流逝而过期。

## 五、验收标准

- [ ] 优才路径生成提醒后，切换至高才通A类 → 优才提醒变为 `archived`，高才通提醒正常显示
- [ ] 提醒列表底部出现「已封存 N 条（切换回原路径后恢复）」折叠区
- [ ] 切回优才路径 → 原优才提醒自动解封为 `active`，高才通提醒变为 `archived`
- [ ] 手动提醒（无 pathway）始终可见，不参与封存
- [ ] 已完成提醒不参与封存
- [ ] 首次选择路径（无旧路径）不触发封存
