# 提醒器路径切换封存 — 产品方案 v1.1（PD评审修正版）

> 2026-05-21 | 优先级 P2 | 2 SP | 评审：agent a86ec9f4

## 一、问题

用户切换身份规划路径后，提醒器混合展示两套路径的提醒。虽有 `pathway` 字段过滤当前路径，但用户无感知——不知道旧提醒去向，也不知道切回去能恢复。

## 二、方案

**切换路径时，旧路径提醒自动封存；切回原路径时自动解封。封存/解封同步至云端。**

### 2.1 触发时机

| 触发             | 位置                         | 备注                                  |
| ---------------- | ---------------------------- | ------------------------------------- |
| 流程控直接选路径 | `process/onSelectDirectPath` | 需在路径覆写前捕获旧路径              |
| 流程控模板选路径 | `process/selectTemplate`     | 独立处理，可能不经 onSelectDirectPath |
| 独立路径选择页   | `path-select/onSelect`       | 同上                                  |

### 2.2 封存逻辑

```
切换路径时（在覆写 selectedPath 之前执行）:
  oldPath = app.globalData.selectedPath || wx.getStorageSync('__selected_path__') || ''
  newPath = id

  if oldPath && oldPath !== newPath:
    所有 pathway === oldPath 且 status 属于 ['pending', 'pending_confirm', 'active', 'deferred', 'chain_waiting'] 的提醒
    → status = 'archived'

    云端同步: cloud.callFunction('reminder-engine', { action: 'archiveByPathway', pathway: oldPath })

切回原路径时:
  所有 pathway === newPath 且 status === 'archived' 的提醒
  → 若 deadline 已过期 > 30 天: status = 'soft_archived'（灰色展示，不计入主列表）
  → 否则: status = 'active'

  云端同步: cloud.callFunction('reminder-engine', { action: 'unarchiveByPathway', pathway: newPath })
```

**关键修正 (PD P0-1/P0-2)：**

- 状态范围从 `['active']` 扩展为 `['pending', 'pending_confirm', 'active', 'deferred', 'chain_waiting']`——云函数生成的提醒初始状态为 `pending`/`pending_confirm`，必须覆盖
- 必须先读取 `oldPath` 再覆写 `selectedPath`，否则永远读不到旧路径
- 解封时对过期 >30 天的提醒标记为 `soft_archived`，防止提醒洪泛

### 2.3 显示规则

| 提醒状态                                        | 展示位置                                            | 视觉             |
| ----------------------------------------------- | --------------------------------------------------- | ---------------- |
| `active`/`pending`/`pending_confirm` + 当前路径 | 主列表                                              | 正常             |
| `active`/`pending` + 无 pathway（手动提醒）     | 主列表                                              | 正常             |
| `archived`                                      | 列表底部折叠区「已封存 N 条（切换回原路径后恢复）」 | 灰色             |
| `soft_archived`（过期>30天）                    | 折叠区内子区「已过期封存 N 条」                     | 浅灰，可一键清除 |
| `completed`                                     | 已完成筛选                                          | 不变             |
| `ignored`                                       | 不展示                                              | 不变             |
| `dismissed`                                     | 不展示，不参与封存                                  | 不变             |

### 2.4 现有过滤逻辑的调整

`loadReminders` 中的 `pathway` 过滤（line 359-365）需增加例外：

```javascript
if (currentPath) {
  formatted = formatted.filter(function (r) {
    return !r.pathway || r.pathway === currentPath || r.status === 'archived';
  });
}
```

### 2.5 不受影响的提醒

- 手动提醒（`pathway` 为空）——始终显示，不封存
- `completed` 状态——不参与封存
- `ignored` / `dismissed` 状态——不参与封存
- `soft_archived` 状态——已是封存终态，不再操作

### 2.6 空状态处理

当所有提醒均被 archived（`stats.total === 0`），在空态文案下追加：
「检测到其他路径的已封存提醒 N 条，切换身份路径后恢复」

### 2.7 里程碑链兼容

封存/解封不自动推进里程碑链（链推进依赖 `completeReminder` 事件触发）。解封后链提醒恢复为解封时的进度——封存期间未完成的操作不会自动补齐。这是预期行为，不视为 Bug。
