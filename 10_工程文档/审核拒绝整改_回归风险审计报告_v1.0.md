### ⚠️ 预存在未部署变更（非本次引入）

`cloudfunctions/feedback-submit/index.js` 存在预变更：新增 `moderateText()` 函数调用 `content-moderation` 云函数进行内容安全审核。

| 风险 | 说明 |
|------|------|
| 当前状态 | 代码已修改但**未部署到CloudBase** |
| 影响 | 如果部署前端代码而不部署此云函数，`feedback-submit` 调用 `content-moderation` 时可能因函数不存在而失败 |
| 失败模式 | `moderateText` 有 try/catch → 返回 `{ pass: false }` → 用户看到"内容安全服务暂不可用" → 安全兜底，拒绝提交 |
| 建议 | 本次部署前，确认是否应同时部署 `feedback-submit` 和 `content-moderation` 云函数 |

## 三、逐文件Bug风险评估

### 1. pages/home/home.js — ⭐ 低风险

| 检查项 | 结果 |
|--------|------|
| 新方法是否覆盖现有方法 | 否 — `toggleConsent`/`enterAsGuest`/`openPrivacyPolicy`/`openUserAgreement` 均为新方法名 |
| setData是否引入新字段冲突 | 否 — `consentChecked` 不与现有字段冲突 |
| 异步竞争 | `handleLogin` 中 `consentChecked` 检查是同步的，在异步登录调用之前，不存在TOCTOU |

| Storage污染 | 游客模式不写Storage ✅ |

### 2. pages/home/home.wxml — ⭐ 低风险

| 检查项 | 结果 |
|--------|------|
| wx:if/wx:elif 条件链是否完整 | `wx:if="{{loading}}"` → `wx:elif="{{unauthenticated}}"` → `wx:elif="{{locked}}"` — 三个互斥条件未变 |
| bindtap 是否指向存在的方法 | `toggleConsent` ✅ `enterAsGuest` ✅ `openPrivacyPolicy` ✅ `openUserAgreement` ✅ |
| catchtap 阻止事件冒泡是否正确 | `catchtap="openPrivacyPolicy"` 和 `catchtap="openUserAgreement"` 用于防止冒泡触发父级 `toggleConsent` — ✅ 正确 |
| disabled 属性语法 | `disabled="{{!consentChecked}}"` — 微信小程序支持 disabled 绑定，语法正确 |

### 3. subpkg-chat/pages/chat/index.wxml — ⭐ 低风险

| 检查项 | 结果 |

| wx:if 是否影响scroll-into-view | 新增 `<view wx:if="{{item.role === 'assistant'}}">` 是气泡内部元素，不影响外层 `id="msg-{{index}}"` 的scroll定位 |

| 流式输出(loading态) | loading气泡是独立 `<view wx:if="{{loading}}">` 块，不在消息遍历中，不受影响 |

### 4. components/floating-ai/floating-ai.wxml — ⭐ 低风险
