## Bug #6: 照片质量校验不生效/不可见 [P0]

**现状**: `utils/ocr.js` 的 `checkImageQuality()` 代码存在（6项检测：分辨率/宽高比/反光/模糊/圆角/倾斜），`add.js` 也调用了，但**用户真机测试反馈没有质量反馈**。可能原因：
- 小程序 `wx.createCanvasContext` 已废弃（基础库 2.9.0+），canvas 取像素可能静默失败
- 质量检测结果虽然 setData 了，但 UI 可能未渲染（条件渲染问题）
- 拍照/相册选择后直接跳到下一步，跳过了质量检测展示步骤

**期望**:
- 质量检测结果必须**可见**——分数条 + 具体问题列表 + 明确通过/不通过
- 不通过时**阻断**下一步，提示用户重新拍摄
- 放弃已废弃的 `wx.createCanvasContext`，改用 `wx.createSelectorQuery` + `Canvas.getContext('2d')` 或纯 JS 像素采样方案
- 确保在 Step 2（图片预览+质量检测）阶段正确展示

**涉及文件**:
- `utils/ocr.js` → `checkImageQuality()` — 改用可用 canvas API
- `pages/documents/add/add.js` — 确保质量检测不通过时阻断流程

## Bug #17: AI Chat 用户画像不得对外透露 + "他是谁"判定逻辑修正 [P0]

**现状**:
1. ai-chat 在回复中直接暴露用户画像信息（如"我看到你正在浏览攻略库，已经选择了优才计划，状态显示为已提交申请等待审批"），这泄露了内部上下文
2. "他是谁"仅用身份状态判断，遗漏了路径选择+职业身份

**期望**:
1. **画像信息只注入 system prompt，不在对话中透露**。AI 应该基于画像调整回答深度和针对性，但永远不能说"根据你的画像，你是XXX状态"。正确做法：直接给出针对该状态的专业回答，不解释为什么知道
2. "他是谁"判定逻辑修正为三维：
   - **身份状态**（userStatus: 未申请/已提交/已获批/永居）
   - **路径选择**（selectedPath: 优才/高才通A/B/C/专才/IANG等）
   - **职业身份**（userSubStatus: 在职人士/学生/企业家/自由职业）
   - 三者叠加才能准确判定用户画像，单靠身份状态不够

**涉及文件**:
- `cloudfunctions/ai-chat/prompts.js` → `buildUserProfile()` — ①加指令"不透露画像信息" ②修正L1判定逻辑
- `cloudfunctions/ai-chat/index.js` — 确保 userProfile 传入包含三维字段

**验证标准**: 调用 ai-chat 传入完整画像后，回复中不应出现任何画像字段原文（如"优才""已提交""攻略库"等）
