### 验证3：CHANGELOG #3 白名单不生效（无效状态值→闸门拒绝通过）

**代码现状**（utils/decision-gate.js）：
```javascript
const VALID_STATUSES = ['unapplied', 'submitted', 'approved', 'permanent']; // line 5

function canMakeDecision() {
  // ...
  if (!userStatus || userStatus === 'skipped') {    // line 29
    return { ok: false, reason: 'identity' };
  }
  return { ok: true, reason: null };                  // line 32
}
```

**发现**：`VALID_STATUSES` 数组已定义，但 `canMakeDecision()` 中**未使用该白名单**进行状态有效性校验。目前仅拦截空值（`!userStatus`）和 `'skipped'` 状态，不拒绝非白名单内的任意字符串（如 `'invalid-status'`、`'legacy-v2'` 等）。

**影响评估**：
- 当前应用层写入 userStatus 的路径有 4 处，分别写入 `'unapplied'`、`'submitted'`、`'approved'`、`'permanent'`、`'skipped'`
- `'skipped'` 已被拦截，其余 4 个值都在 VALID_STATUSES 中
- **生产环境风险：低**。仅当数据存储被污染（旧版本脏数据、storage 手动篡改）时才会触发
- **代码完整度：中**。白名单已声明但未接线到判断逻辑

**建议**：V4.3 中补充 VALID_STATUSES 校验逻辑：
```javascript
if (!userStatus || userStatus === 'skipped' || !VALID_STATUSES.includes(userStatus)) {
    return { ok: false, reason: 'identity' };
}
```

**结论**：⚠️ 条件通过。VALUD_STATUSES 已定义但未接线；当前生产环境下无效状态值仅可能来自 storage 污染，实际触发概率极低。**不阻塞上传，列入 V4.3 技术债务。**

### 验证5：支付全链路（登录→攻略书购买→wx.requestPayment→回调→解锁）

**溯源路径**：
```
1. 闸门检查
   guidebooks/index.js:555 → canMakeDecision() ✅

2. 创建支付订单（云函数）
   guidebooks/index.js:574-578 → wx.cloud.callFunction('payment', { action: 'unlockAllPhases' })

3. 拉起微信支付
   guidebooks/index.js:588-593 → wx.requestPayment({ timeStamp, nonceStr, package, signType, paySign })

4. 支付成功回调 → 立即解锁本地状态
   guidebooks/index.js:597 → app.globalData.guidebookAllUnlocked = true

5. 云端确认支付
   guidebooks/index.js:598-601 → callFunction('payment', { action: 'confirmPayment', orderId })

6. 确认成功 → 刷新页面
   guidebooks/index.js:605-606 → wx.showToast('全部关卡已解锁！') + self.init()

7. 确认失败降级 → 延迟刷新
   guidebooks/index.js:608-610 → setTimeout(() => self.init(), 1500) + Toast '订单确认中'
```

**支付异常处理**：
- 用户取消支付：`fail` 回调中 `errMsg.indexOf('cancel')` 判断，静默不提示
- 其他支付失败：`wx.showToast({ title: '支付失败，请重试' })`
- 云端确认失败：降级延迟刷新，不丢失解锁状态（本地 unlockAll 已先写入）

**结论**：✅ 通过。支付全链路完整——闸门→下单→支付→本地解锁→云端确认→刷新，异常分支均有降级处理。`guidebookAllUnlocked` 在 `wx.requestPayment` success 回调中先写入本地，即使云端 confirmPayment 失败也不影响用户侧功能。
