## 二、部署顺序（严格执行）

```
1. 备份: 导出当前云函数代码 → 本地存档
2. 配置: CloudBase 控制台 → 环境变量 → WECOM_WEBHOOK_URL
3. 部署 cf-alert: npm install → tcb fn deploy cf-alert --http-trigger /cf-alert
4. 冒烟 cf-alert: curl GET /cf-alert/status → 200 + { checkedAt, last24h }
5. 部署业务云函数(5个) 逐台: npm install → tcb fn deploy xxx
6. 冒烟每台: invoke → ErrMsg="" / Duration>0ms
7. 部署小程序: 微信开发者工具 → 上传 → 提交审核
8. 冒烟小程序: 真机 → 兑换码流程 → 攻略书解锁 → 进度条更新
```

### 回滚顺序（逆序）
```
1. 云函数: tcb fn deploy xxx --source=备份包
2. 小程序: 微信后台 → 版本管理 → 回退到上一版本
3. cf-alert: 删除 HTTP 触发器 → 删除云函数
```

## 六、应急预案

### 场景A: 云函数部署后崩溃
- **现象**: 用户端"网络异常" / CloudBase 日志 Duration=0ms
- **排查**: CloudBase → 云函数 → 日志 → 搜索 "Cannot find module" 或 ReferenceError
- **处置**: 1) 回滚到备份包 2) 本地 npm install 确认依赖完整 3) 重新部署

### 场景B: 企微告警风暴
- **现象**: 群消息刷屏
- **排查**: 检查 cf_error_logs 集合中同一 fnName+action 记录数
- **处置**: 1) 删除 WECOM_WEBHOOK_URL 环境变量（立即静默） 2) 排查根因后恢复

### 场景C: 攻略书仍不全部解锁
- **现象**: ZGB-6B93CBC3 同类型用户反馈未全解锁
- **排查**: 检查该用户 CloudBase users.membershipLevel → 若为 free → 确认 invite-code 未被正确兑换
- **处置**: 1) 运营后台手动设置 membershipLevel='basic' 2) 通知用户清除小程序缓存后重试

### 场景D: cf_error_logs 写入失败
- **现象**: cf-alert/status 返回 last24h.totalErrors=0 但已知有错误
- **排查**: CloudBase → 数据库 → cf_error_logs → 检查安全规则（是否禁止写入）
- **处置**: 1) 数据库权限 → 仅创建者可写 2) 若为 SDK 版本问题 → 回滚 cf-alert
