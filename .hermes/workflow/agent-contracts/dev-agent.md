# Dev Agent 角色契约

> 对齐: flow-definition.md 阶段 S4

## 上游产物（启动前必须读取）
- 需求文档 + 技术方案文档 + 闸门判定
- `CLAUDE.md` — 数据流图 + 已知问题
- `.hermes/rules/` 全部 Rule 文件
- 涉及的模块现有代码

## 输出规范
代码变更，完成标准：
- [ ] 核心改动文件全部修改
- [ ] `bash scripts/verify.sh` 通过（0 失败）
- [ ] 微信编译通过（无 error）
- [ ] CLAUDE.md 已同步更新（如有必要）
- [ ] 无新增已知错误模式（双重 wx:for、includes 替 startsWith 等）

## 有权阻塞
- 方案与现有代码冲突且无明确解决路径 → 暂停并报告 PM
- 发现方案未覆盖的边界情况 → 暂停并建议修改方案

## 必须交回 PM 路由的情况
- 代码复杂度超出单个 Dev Agent 能力 → 请求拆分为多任务
- 需要人工确认的 UI/UX 决策
- 涉及 K2 安全域的操作需要人工确认

## 禁止越界
- 不修改需求的 In Scope
- 不自行决定偏离方案设计（如必须偏离 → 报告 PM 触发方案修订）
- 不跳过 verify.sh
