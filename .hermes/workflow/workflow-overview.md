---

## 一分钟理解 Harness

Harness 是一套「AI 驱动的软件工程约束系统」。它把开发流程切成了七站，每站一个 AI Agent 负责，站与站之间有明确的交接规则——就像工厂流水线。

**核心原则**：AI 说"做好了"不算，verify.sh 通过才算。

## 研发链路全貌

```
你（人类 PM / 琅琊）
  │
  │  需求描述（口头/文档/看板）
  ▼
┌─────────────────────────────────────────────────────┐
│               PM Agent (路由调度)                     │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ S1 需求   │→│ S2 方案   │→│ S3 闸门   │          │
│  │ 分析     │  │ 设计     │  │ 总控     │          │
│  └──────────┘  └──────────┘  └────┬─────┘          │
│                                   │ 准入            │
│  ┌──────────┐  ┌──────────┐  ┌───▼──────┐          │
│  │ S6 测试   │←│ S5 审查   │←│ S4 开发   │          │
│  │ 验证     │  │ 代码     │  │ 实现     │          │
│  └────┬─────┘  └──────────┘  └──────────┘          │
│       │                                             │
│       ▼                                             │
│  ┌──────────┐                                       │
│  │ S7 交付   │──→ 你的审批                           │
│  └──────────┘                                       │
└─────────────────────────────────────────────────────┘
```

### 每站做什么

| 站 | Agent | 输入 | 输出 | 你能看什么 |
|----|-------|------|------|-----------|
| S1 | 需求分析 | 你的需求描述 | 结构化需求文档 | 确认边界对不对 |
| S2 | 方案设计 | 需求文档 | 技术方案+影响文件 | 确认方向对不对 |
| S3 | 闸门总控 | 需求+方案 | 准入/打回 | 确认能不能做 |
| S4 | 开发实现 | 需求+方案 | 代码+verify.sh | 看代码改动 |
| S5 | 代码审查 | 代码 diff | 审查报告 | 看有没有偏离需求 |
| S6 | 测试验证 | 代码+审查 | 测试报告 | 看功能对不对 |
| S7 | 交付 | 全部产物 | 交付结论 | **最终确认** |

## 你会参与什么

### 1. 提需求时
告诉 PM Agent 你想做什么。它会路由给需求分析 Agent。
你可以直接说"在证件夹加一个新卡槽"，Agent 会自动拆解。

### 2. 看方案时
方案设计 Agent 会列出所有受影响的文件。
你在这一步可以纠正方向——说"这个文件不要动"或"用另一个云函数"。

### 3. 开发完成时
运行 `bash scripts/verify.sh` 看门禁结果。
通过（0 失败）→ 可以交付。
不通过 → 退回 Dev Agent 修复。

### 4. 流程卡住时
PM Agent 会暂停并告诉你阻塞点。
例如："需求边界不清晰，请确认 X 功能是否在本次范围内。"
你的角色是**做决策**，不是写代码。

## 日常命令

```bash
# 开发完成后
bash scripts/verify.sh

# 建立基线（首次/重大变更后）
bash scripts/verify.sh --baseline

# 对比基线（发现新增失败）
bash scripts/verify.sh --diff

# 检查工程资产完整性
bash scripts/workflow-verify.sh
```
