# AI Chat 防误读控制规范

> 规则类型: P0 硬约束 (必须注入 AI Chat system prompt)
> 适用于: ai-chat 云函数、指引牌模块、所有面向用户的 AI 对话

## 三层知识域隔离

| 层级 | 定义 | 举例 | 检索权限 |
|:----:|------|------|:--------:|
| **K0** | 用户可见知识 | 文档名称、签发机构、适用路径 | 可自由检索和引用 |
| **K1** | 条件可见知识 | 材料标准、有效期、归档策略 | 可检索，须加免责标注 |
| **K2** | 系统内部知识 | 视觉特征、校验规则、安全机制 | **禁止进入 RAG 检索** |

## 六条禁止响应规则 (注入 system prompt)

下列行为在任何情况下都不允许：

1. **禁描述防伪特征** — 不得描述证件的物理防伪特征（水印/镭射/荧光纤维/安全线等）
2. **禁做真伪判断** — 不得对用户上传的材料做真伪鉴定或给出"可能是假的"类判断
3. **禁透露技术实现** — 不得描述 OCR 引擎的识别逻辑、校验算法、特征匹配方式
4. **禁包装校验规则** — 不得将内部的校验规则包装为"官方标准"或"你应该检查"类建议
5. **禁暴露识别能力** — 不得主动告知用户可以识别哪些证件类型及其检测维度
6. **禁输出 K2 字段** — 不得在回答中引用任何 K2 级字段的规格、阈值、格式定义

## RAG 三层过滤

1. **入库前过滤** — K2 来源的数据不写入 knowledge_chunks
2. **检索时过滤** — 检索结果排除 K2 标签的 chunk
3. **检索后二次过滤** — FORBIDDEN_PATTERNS 正则扫描，命中则从结果中移除

### FORBIDDEN_PATTERNS
```
防伪特征|水印|镭射|荧光|安全线|紫外|红外|微缩文字|
真伪|假证|伪造|PS|篡改|拼接|痕迹|
OCR引擎|识别算法|匹配阈值|特征提取|校验位
```

## 安全横幅触发条件

当用户消息匹配以下模式时，自动在回答中插入安全横幅：
- 询问"这个证件是真的吗"
- 询问"怎么看真假"
- 询问"你们能检测出来吗"
- 上传图片 + 任何真伪相关关键词

横幅内容: "住港伴仅提供材料准备指引，不对证件真伪做判断。如对证件有效性有疑问，请咨询入境事务处。"

## 日志要求

- 所有 AI Chat 回答记录 `safety_rules_triggered` 字段
- K2 扫描命中记录 `k2_leak_flagged` 字段
- 安全横幅展示记录 `safety_banner_shown` 字段
