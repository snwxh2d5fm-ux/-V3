# 数据处理管线规范

> 规则类型: 硬约束 (所有入库数据必须遵循)
> 适用于: 所有批次数据处理、知识库建设、数据库写入操作

## 核心规则

### 1. 原数据与处理数据严格区隔
- 原数据锁定后不得直接引用
- 所有产品引用必须经过三道流程处理
- 详见 Skill: `data-cleaning-run`

### 2. 三道路程（原则）
- **第一程-清洗**: 脱敏 → 去营销 → 标准化 → 去重
- **第二程-校验**: 事实核查 → 跨源一致性 → 时效性 → 追溯链
- **第三程-评估打标**: 绿/黄/红 → 置信度五级 → 来源可靠性 → K0/K1/K2

### 3. 入库协议
- 统一 JSONL 格式，每行一条记录
- 必填字段: `_id`, `contentHash`, `source`, `sourceType`, `knowledgeDomain`, `confidence`, `grade`, `processedAt`
- K2 级数据不得入知识库（仅供内部参考）

### 4. 禁止事项
- 禁止跨库引用（产品引用只能来自处理后的知识库）
- 禁止引入夸大、虚假、营销内容
- 禁止将 K2 级数据写入 knowledge_chunks
- 杭州入园等跨域语料仅萃取方法论，不入库实务信息

## 批次命名规范
- 格式: `EX-YYYY-MMDD-NNN`
- 示例: `EX-2026-0507-001`
