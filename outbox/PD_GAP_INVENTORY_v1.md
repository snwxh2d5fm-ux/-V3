## P0-1: 攻略书v6对齐

### 现状

- 攻略书系统当前为 v4.5（数据层 V3 双驱动推荐 + v4.5 inline 解析器）
- 47 篇文章，10 个分类，13 条路径覆盖
- Tab4「攻略精选」WXML 渲染块缺失（P0-3 已修复于 29e82cb）
- 推荐算法本地 V3（STATE_PROFILE + PATH_TAGS）但云端 V1（knowledgeDomain 权重），结果跳变

### v6 目标

| 维度 | 当前 (v4.5) | v6 目标 |
|------|------------|---------|
| 推荐引擎 | V3 双驱动（状态×路径） | V6 三驱动：状态×路径×**行为**（阅读/收藏/分享权重） |
| 分类体系 | 10 类硬编码 | 动态分类 + 标签云 + 自动归类 |
| 内容格式 | Markdown → WXML 解析 | 富媒体（图/表/交互卡片）+ 多语言版本切换 |
| 搜索 | 无 | 全文搜索 + tags 过滤 |
| 个性化 | 仅路径+状态 | 阅读历史 → 继续阅读 → 相关推荐链 |
| 数据同步 | 云端 V1 与本地 V3 不一致 | 统一 V6 算法 + CloudBase 实时同步 |
| 内容管理 | 手动编码 | CMS 面板 → 文章 CRUD → 自动发布 |

### 缺口根因

1. **推荐算法分裂**：本地 guidebook-data.js V3 vs 云端 guidebook/ V1，云同步后推荐跳变
2. **WXML 渲染块**：Tab4 曾完全空白（已修），但仅简单列表无交互卡片
3. **搜索缺失**：47 篇文章无搜索入口，用户靠翻找
4. **行为数据空白**：无埋点 → 推荐无法利用行为信号

### 建议修复路径

```
Phase A (本周): 云端 guidebook/index.js 升级到 V3（同步 PATH_TAGS + STATE_PROFILE）
Phase B (下周): 全文搜索 + tags 过滤
Phase C (2周): 埋点接入 → V6 三驱动推荐
Phase D (3周): 富媒体卡片 + CMS 面板
```

## P0-2: 信息栏数据活水

### 现状

`pages/info/index/index.js` — **3 条硬编码更新 + 2 条硬编码解读**，无任何数据源：

```javascript
updates: [
  { id: '1', title: '入境处更新合资格大学名单', date: '2026-05-01', ... },
  { id: '2', title: '优才计划审批周期调整', date: '2026-04-20', ... },
  { id: '3', title: '高才通续签审核标准趋严', date: '2026-04-15', ... }
]
```

### 缺口

| 问题 | 影响 |
|------|------|
| **完全静态** | 3 条数据永远不会更新，用户看到过时信息 |
| **无数据源** | 没有接入任何政策源（入境处/劳工局 RSS/API） |
| **无推送** | 政策变更不通知用户，产品失去"信息差"核心价值 |
| **无个性化** | 所有用户看同样信息，与选择的路径无关 |
| **页面入口孤立** | info 页面不在 TabBar，不在主要流程中，入口难找 |

### 目标

信息栏应成为产品的 **"政策信息差引擎"** — 这是香港身份产品的核心差异化价值：

1. **数据源接入**：入境处公告 / 劳工及福利局 / 教育局 RSS + 人工精选
2. **政策 → 路径映射**：每条更新标注 affected paths，按用户路径过滤推送
3. **订阅推送**：微信服务通知 / 模板消息
4. **解读社区**：UGC 解读 + AI 解读双轨
5. **入口强化**：首页信息流 / 攻略书内嵌 / 流程控时间线关联

### 建议修复路径

```
Phase A (本周): 接入 2 个政策 RSS 源 → CloudBase 定时拉取 → info 页面动态渲染
Phase B (下周): affected-paths 标注 + 按路径过滤
Phase C (2周): 模板消息推送（政策变更通知）
Phase D (3周): AI 解读 + UGC 解读层
```

## P2-1: Onboarding 引导

### 现状

新用户流程：首页 → 登录 → 状态选择页 → 流程控 Tab

**缺失的引导环节**：

| 缺口 | 现状 |
|------|------|
| **欢迎引导** | 无。登录后直接跳到状态选择，无产品价值介绍 |
| **路径选择辅助** | status-select 页仅简单选项列表，无对比/推荐/测评 |
| **首次任务引导** | 无。用户到了流程控后不知该做什么 |
| **功能发现** | 5 个 Tab 无引导说明，用户不知道每个 Tab 做什么 |
| **成就/进度感知** | 无。用户没有"完成了什么、下一步是什么"的感知 |

### 目标

Onboarding 应该是 **3 步递进式**：

```
Step 1: 你是谁 → 路径快速测评（3-5 题）→ 推荐路径
Step 2: 你到哪了 → 选择当前阶段 → 生成初始时间线
Step 3: 开始吧 → 首个任务卡片 → "完成证件照上传" → 解锁下一步
```

### 建议修复路径

```
Phase A (本周): 快速测评（路径推荐引擎 — 前端 5 题版）
Phase B (下周): 首次任务引导 + 功能导览浮层
Phase C (2周): 阶段选择 → 自动生成时间线 + 首个任务
```

## P2-2: 付费锁屏策略

### 现状

- 会员体系已搭建：3 档（Basic/Pro/Premium），¥39.90/¥299.90/¥699.90 月付
- 支付流程完整：createOrder → wx.requestPayment → confirmPayment
- 锁屏入口单一：仅 home.js 的 `isLocked` 检查 → 跳转会员页

### 缺口

| 问题 | 影响 |
|------|------|
| **无免费额度耗尽机制** | FREE_LIMITS 定义了但多数功能无实际限制 |
| **无价值感知锁屏** | 用户没体验过价值就被锁 → 转化率低 |
| **锁屏触发点单一** | 只在首页检查，功能内无渐进式锁屏 |
| **无试用期结束提示链** | 180 天试用但无到期前提醒（7天/3天/当天） |
| **无社交裂变解锁** | 无可分享解锁/邀请得时长的机制 |
| **无差异化锁屏策略** | 所有功能锁屏一致，无"核心功能免费 + 高级功能付费"分层 |

### 目标付费策略

```
免费层 (Free):
  ✅ 路径选择 + 基础测评
  ✅ 首条规则链提醒
  ✅ 3 篇攻略书/月
  ✅ 证件夹基础功能（5 个证件）
  ❌ AI 对话（3 次/天后锁）
  ❌ 高级推荐（仅路径匹配，无行为驱动）
  ❌ 政策推送

会员层 (Pro):
  ✅ 全部免费功能
  ✅ 无限 AI 对话
  ✅ 全部攻略书 + 个性化推荐
  ✅ 全部规则链提醒
  ✅ 政策推送 + 日历同步
  ✅ 无限证件 + OCR 批量识别
  ✅ AI 材料生成
```

### 建议修复路径

```
Phase A (本周): 免费层限制实际生效（AI 对话 3 次/天 + 攻略书 3 篇/月）
Phase B (下周): 价值感知锁屏 — 功能使用到阈值时弹窗提示升级
Phase C (2周): 试用到期提醒链（30天/7天/3天/当天模板消息）
Phase D (3周): 社交裂变 — 邀请得 7 天 Pro 试用
```
